Introduction

Git is a version control system for tracking changes in code. It was created by linus Torvalde in 2005, and has been maintained by Junio Hamano since then.

Git Capabilities

  • Manage projects with Repositories
  • Clone a project to work on a local copy
  • Control and track changes with Staging and Committing
  • Branch and Merge to allow for work on different parts and versions of a project
  • Pull the latest version of the project to a local copy
  • Push local updates to the main project

Using Git

  • Create a Repository by initializing Git in a project folder
  • A hidden folder is created to track changes in the repository folder
  • Any files with changes or newly created files are considered modified
  • You select modified files that you want to Stage
  • Staged files are Committed, which prompts Git to store a permanent snapshot of the files
  • A full history of every commit can be viewed
  • You can revert back to any previous commit
  • Git does not store a separate copy of every file in every commit, but keeps track of changes made in each commit

Git vs. GitHub

Git is not the same as GitHub. GitHub is an online/remote repository which makes tools that use Git. GitHub is very popular, and is the largest host of source code in the world. However, Git can be used with other remote repositories (like Bitbucket and GitLab) as well. GitHub has been owned by Microsoft since 2018.

Using Git with Command Line

Git is used through the Command Shell or Terminal. For Windows, you can use Git bash, which comes included in Git for Windows. For Mac and Linux, you can use the built-in terminal. The first step is to verify that Git was properly installed by entering the following command.

[user@localhost] $ git --version

If Git is installed, it should show something like git version X.Y

Configure Git

Set up your username and email as follows. This information is recorded with each commit. It is also helpful to register on GItHub with the same username and email in order to work with GitHub later on.

[user@localhost] $ git config --global user.name "your.user.name"

[user@localhost] $ git config --global user.email "you@yourEmail.com"

Note: Use global to set username/email for all repositories on your computer. Remove global to set username/email only for the current repository.

Creating Git Folder

Create a new folder for your project.

[user@localhost] $ mkdir newProject

[user@localhost] $ cd newProject

Note:
mkdir makes a new directory.
cd changes the current working directory.

If you already have a folder/directory you would like to use for Git, navigate to it in the command line, and initialize Git.

Initialize Git

Once you are in the correct folder, you can initialize Git with the following.

[user@localhost] $ git init

Git creates a hidden folder to keep track of changes in the folder you initiated it on.

Staging New Files

Once a repository folder has been initialized, the folder contents then need to be staged in order for Git to track changes to existing files/sub-folders or new file/sub-folder additions. Prior to staging, we can check the current Git status in the repository as follows.

[user@localhost] $ git status

On branch master

No commits yet

Untracked files:
(use "git add ..." to include in what will be committed)
your_file1.html
your_file2.css

nothing added to commit but untracked files present (use "git add" to track)

The above response shows that Git is aware of two untracked files, your_file1.html and your_file2.css. Files in a Git repository folder can be in one of two states.

  • Tracked - files that Git knows about and are added to the repository
  • Untracked - files that are in your working directory, but not added to the repository

Files are initially untracked when they are first added to a working directory that is also a Git repository. To get Git to track them, they have to be staged i.e. added to the staging environment. This can be done as follows.

[user@localhost] $ git add your_file1.html your_file2.html

or

[user@localhost] $ git add --all

or

[user@localhost] $ git add --A

Note: The shorthand command for git add --all is git add --A

Git Commit

Once files have been staged i.e. added and thus changes are being tracked, the next step is to commit changes. A commit is a save point in the project, that can be reverted to if a bug is discovered or some other changes need to be made.

When a commit is made, a descriptive commit message should always be included to summarize what has changed and when. A commit with a message is done as follows.

[user@localhost] $ git commit -m "First commit of Your Project"

[master (root-commit) 221ec6e] First commit of Your Project
2 files changed, 55 insertions(+)
create mode your_file1.html
create mode your_file2.css
